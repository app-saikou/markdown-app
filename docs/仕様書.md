# 思考構造化アプリ 要件定義（MVP）

## 開発方針

本アプリは **React Native × Expo** を用いて開発する。

※本要件定義では技術詳細には触れない。

---

# 1. 目的

思いつきやアプリアイデアを素早く記録し、構造化し、MarkdownおよびPDF形式でPCに取り込める形へ変換する。

本アプリは「Markdown作成ツール」ではなく、
**思考を開発素材へ変換するための構造化ツール**である。

---

# 2. 想定ユースケース

- 移動中にアプリアイデアが浮かぶ
- 音声またはテキストで雑に記録する
- AIで構造化する
- MarkdownまたはPDF形式で書き出す
- PC（例：Cursor）に取り込み、実装・Issue作成・ナレッジ蓄積に活用する
- 企画資料としてPDFを共有する

---

# 3. コアオブジェクト

## 3.1 Capture（キャプチャ）

思考の生データ。

### 属性

- rawText
- createdAt
- sourceType（voice / text）

### 要件

- 起動後すぐ入力できる
- 音声入力が可能
- 生データは常に保持される
- 自動保存される

---

## 3.2 Idea（構造化された思考）

Captureを整理した単位。

### 属性

- title
- summary
- sections（構造化されたブロック群）
- tags
- status（draft / structured / exported）

### 要件

- Captureから生成できる
- 手動編集可能
- 構造化案は「提案」として表示される
- 採用／破棄を選択できる

---

## 3.3 Structure（アウトライン）

出力の骨子。

### 属性

- 見出し階層
- セクション順序
- セクションタイプ

### 要件

- テンプレートに基づいて生成される
- 並び替え可能
- 折りたたみ可能
- MarkdownおよびPDF出力の元データとなる

---

## 3.4 Template（構造テンプレート）

用途別の構造定義。

### 初期テンプレ例

- アプリ案
- 技術検討メモ
- UI改善案
- 課題分析

### 要件

- 構造化時に選択できる
- 将来的に追加可能

---

## 3.5 Export（出力）

外部利用用の最終形式。

### 出力形式

- Markdown（.md）
- PDF（.pdf）

### 要件

- ワンタップコピー（Markdown）
- ファイル共有（Markdown / PDF）
- 常に再生成可能（編集内容から自動生成）
- PDFは資料としてそのまま共有可能な体裁を持つ

---

# 4. 主要フロー

## 4.1 即時キャプチャ

1. 新規作成
2. 音声またはテキスト入力
3. 自動保存

---

## 4.2 構造化

1. テンプレート選択
2. AIによる構造化提案
3. 採用または修正
4. Ideaとして保存

---

## 4.3 出力

1. プレビュー確認（Markdown / PDF）
2. コピーまたは共有
3. ステータスを「exported」に変更

---

# 5. UX要件

## 5.1 スピード重視

- 思考記録までの操作は最小ステップ
- 編集よりも「捕まえる」ことを優先

## 5.2 意味操作中心

- 記法操作より構造操作を優先
- セクション単位で編集可能
- セクションタイプ変更が簡単

## 5.3 片手操作前提

- 主要アクションは親指到達圏内
- 構造化・出力は1アクションで可能

---

# 6. 保存要件

- ページ単位で保存
- 検索可能（タイトル / 本文）
- 並び替え可能（更新日 / ステータス）

---

# 7. 非目的（MVPでは行わない）

- 高度なMarkdown装飾編集
- クラウド同期
- チーム共有機能
- リッチテキスト編集

---

# 8. 成功指標

- 思いつきから出力まで3分以内で完了できる
- コピペ往復が不要になる
- PC側でそのまま開発素材として利用できる
- PDFとしてそのまま企画資料共有が可能

---

# 9. 本質的価値

- 思考を逃さない
- 構造を与える
- 開発や共有に直結する形式へ変換する

この3点を満たすことが本アプリの成立条件である。
