# AIモード サンプル集

## 整理する（structure）

ユーザーの言葉はそのままに、構造だけ整える。

---

### サンプル1：会議メモ

**インプット（雑なメモ）**
```
今日の打ち合わせ
デザインのリリースは来週月曜
バックエンドAPIは山田さんが担当、今週中に仕様書
テストは木曜から、QAは鈴木さんに依頼する
次回打ち合わせは金曜15時
懸念点：本番環境の権限まだ取れてない
```

**アウトプット（期待値）**
```markdown
# 打ち合わせメモ

## スケジュール

- デザインリリース：来週月曜
- バックエンドAPI仕様書：今週中（担当：山田さん）
- テスト開始：木曜〜（QA担当：鈴木さん）
- 次回打ち合わせ：金曜 15:00

## 懸念点

- 本番環境の権限がまだ取れていない
```

## 発展させる（expand）

インプットを起点に、AIが関連視点・具体案・論理展開を追加する。

---

### サンプル1：ビジネスアイデアの種

**インプット**
```
犬の散歩代行アプリ作りたい
近所の人に頼める感じ
信頼性が大事
```

**アウトプット（期待値）**
```markdown
# 犬の散歩代行マッチングアプリ

## コアバリュー

「近所の信頼できる人に、安心して愛犬を預けられる」

## ターゲット

- **依頼側**：共働き・急な外出が多いペットオーナー
- **受け手側**：副収入を得たい近隣住民・動物好きな人

## 差別化ポイント

- 近所限定のマッチング（顔が見える安心感）
- 散歩中のGPSトラッキング・写真通知
- レビュー・リピート機能で信頼を可視化

## 懸念点と対策

| 懸念 | 対策 |
|------|------|
| トラブル時の責任 | 保険オプションの提供 |
| 集客 | 地域SNS・動物病院との提携 |
| 収益化 | 手数料15〜20%、プレミアム機能課金 |

## 次のアクション

- [ ] 競合調査（Wag!, Rover等）
- [ ] ペットオーナーへのヒアリング（5人）
- [ ] MVPの機能定義
```

### サンプル2：学習メモの発展

**インプット**
```
マーケティングの本読んだ
人は機能より感情で買うらしい
プロダクトの話より体験の話をしろって
```

**アウトプット（期待値）**
```markdown
# 感情購買と体験マーケティング

## 核心メッセージ

> 人はスペックではなく、「どう感じるか」で買う。

## なぜ感情が重要か

- 購買決定の95%は無意識（感情・直感）に基づく（Harvard Business School）
- 機能訴求は比較されやすく、価格競争に陥りやすい
- 体験訴求は差別化しやすく、ブランドロイヤルティを生む

## 実践への応用

### Before（機能訴求）
「バッテリーが3000mAh、軽量150g」

### After（体験訴求）
「一日中充電を気にせず、ポケットに入れても気にならない」

## アクションに落とす

- [ ] 自社プロダクトのコピーを「体験」視点で書き直す
- [ ] ユーザーインタビューで「どんな気持ちになった？」を聞く
- [ ] SNS投稿を機能説明→シーン描写に変える
```
